## Architecture

This is a NixOS + Home Manager configuration using **flake-parts** as the module framework and **import-tree** to auto-discover all `.nix` files under `config/`.

### Key structural concepts

- **`flake.nix`** is auto-generated by `flake-file` â€” do not edit it directly. Run `nix run .#write-flake` to regenerate it after any `flake-file.inputs` change.
- **`flake-file`** enables co-located input declaration: each `.nix` file declares the flake inputs it needs via `flake-file.inputs = { ... }` right alongside the code that uses them. For example, `claude-code` is declared in `config/users/joshua/default.nix`, `home-manager` in `config/modules/home-manager.nix`, and `nvf` in `config/modules/terminal/neovim/neovim.nix`. `commonHost.nix` only declares the core framework inputs (nixpkgs, flake-parts, flake-file, import-tree) and the flake outputs definition.
- **`import-tree`** recursively imports every `.nix` file under `config/`, so adding a new file is enough to include it â€” no manual `imports` wiring needed at the flake level.
- **`flake-parts` module system**: each file contributes to the flake via `flake.*` attributes (`flake.nixosModules`, `flake.homeModules`, `flake.nixosConfigurations`, etc.). These are merged across all imported files.

### Config layout

| Path | Purpose |
|---|---|
| `config/hosts/` | Per-host NixOS configurations and hardware; `commonHost.nix` defines shared host settings and core flake-file inputs/outputs |
| `config/modules/` | Reusable NixOS/Home Manager modules (git, SSH, neovim, etc.) |
| `config/profiles/` | Profile modules that group other modules (`profilesCommon` imports git, SSH, neovim) |
| `config/users/joshua/` | User account definition and home-manager configuration |

### Module naming convention

- `flake.nixosModules.<name>` â€” NixOS system-level modules (used in `nixosSystem { modules = [...] }`)
- `flake.homeModules.<name>` â€” Home Manager modules (used under `home-manager.users.<user>.imports`)
- `import-tree` derives the module name from the file path, but explicit names are assigned inside each file using `flake.nixosModules.<name> = ...`

### Neovim

Neovim is configured via **nvf** (home manager module at `inputs.nvf.homeManagerModules.default`). Config is split across:
- `neovim.nix` â€” top-level nvf enable/alias/theme settings and imports
- `neovimOptions.nix`, `neovimKeybinds.nix`, `neovimLsp.nix`, `neovimPlugins.nix`, `neovimAutocmds.nix` â€” each adds to `programs.nvf.settings.vim.*`

### Adding a new host

1. Create `config/hosts/<hostname>/` with `default.nix`, `configuration.nix`, and `hardware-configuration.nix`
2. The `default.nix` should define `flake.nixosConfigurations.<hostname>` referencing the relevant `nixosModules`
3. `import-tree` picks it up automatically

### Adding a new module

Create a `.nix` file anywhere under `config/modules/` that exports `flake.nixosModules.<name>` or `flake.homeModules.<name>`, then import it from a profile or host module. If the module needs a new flake input, declare it with `flake-file.inputs = { ... }` in that same file, then run `nix run .#write-flake` to regenerate `flake.nix`.
